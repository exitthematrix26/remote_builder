Key KEDA config (generate this exactly):
apiVersion: keda.sh/v1alpha1
kind: ScaledObject
metadata:
  name: rbe-workers-{{ .Values.tenant.name }}
  namespace: tenant-{{ .Values.tenant.name }}
spec:
  scaleTargetRef:
    apiVersion: argoproj.io/v1alpha1
    kind: Rollout # targets Argo Rollout, not Deployment
    name: rbe-workers
  minReplicaCount: 0 # scale to zero when idle
  maxReplicaCount: {{ .Values.worker.maxReplicas }}
  cooldownPeriod: 60
  triggers:
    - type: prometheus
      metadata:
        serverAddress: http://prometheus-operated.cluster-infra:9090
        metricName: grpc_requests_per_minute
        query: |
          sum(rate(grpc_server_handled_total{
            namespace="tenant-{{ .Values.tenant.name }}",
            grpc_service="build.bazel.remote.execution.v2.Execution"
          }[2m])) * 60
        threshold: "5" # one worker pod per 5 RPM
Observable outcome: Run load generator → watch kubectl get pods -n tenant-acme -w

kind: ApplicationSet
metadata:
  name: rbe-tenants
  namespace: argocd
spec:
  generators:
    - git:
        repoURL: https://github.com/YOUR_ORG/rbe-lab
        revision: HEAD
        directories:

Key Architecture Principles to Enforce in All Generated Code
Terraform / Argo CD boundary is sacred. Terraform creates the cluster, node groups,
IAM, S3, ECR. It never touches K8s manifests. Argo CD owns everything inside the cluster.
Never use the Terraform K8s provider for workload resources.
Workers must be stateless. No local state on worker pods. All build outputs go to
MinIO/S3 (CAS). Any worker can die mid-action — Bazel will retry. This is why Spot works.
PodDisruptionBudgets on workers. Even though workers are stateless, a PDB should
ensure at least N-1 workers survive a node drain during active builds.
Use Argo Rollout, not Deployment, for workers. This is required for canary support
via Istio. The KEDA ScaledObject must target argoproj.io/v1alpha1/Rollout, not apps/v1/Deployment.
Scale signal is gRPC RPM, not CPU. Workers at idle have near-zero CPU. KEDA must
use the Prometheus metric, not the default CPU HPA. This was already validated in a prior
lab and confirmed correct.
Per-tenant ResourceQuotas are non-negotiable. Define them in the Helm chart defaults.
One tenant's burst must never affect another tenant's workers.
MinIO is a drop-in for S3. The Buildbarn CAS config should use the S3 backend pointed
at MinIO locally. In production, change the endpoint URL and credentials. No chart changes.
GitHub is the source of truth. If it is not in Git, it does not exist. Never run
kubectl apply manually against the cluster for anything Argo CD manages. The only
exception is the initial Argo CD bootstrap (chicken-and-egg).
